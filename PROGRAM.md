<architecture>
Qwen3.5-2B is a HYBRID model -- not a standard transformer:
  24 layers = 6 x (3 x Gated DeltaNet -> FFN + 1 x Gated Attention -> FFN)
  Only 6/24 layers are standard attention (q_proj, k_proj, v_proj, o_proj).
  The other 18 layers are Gated DeltaNet with different projection names.

LoRA baseline targets only: ["q_proj", "v_proj"] in the 6 attention layers.
Before targeting DeltaNet layers, WebSearch "Qwen3.5 LoRA target modules deltanet"
to find the correct parameter names.
</architecture>

<exploration_tiers>
Work top-down. Do not use generic data (Tier C) until Tier A+B are exhausted.

TIER 1 -- Constraint-matched data (highest priority -- start here)
  data_rejection      : TRAIN_DATA = "data/train_ifeval_rejection.jsonl"
                        Train directly on IFEval prompts with verified passing responses.
  data_synthetic      : TRAIN_DATA = "data/train_synthetic.jsonl"
                        Train on programmatically generated constraint-satisfying examples.
  data_mixed_ab       : Combine rejection + synthetic in one training run.
                        Concatenate the two files or alternate batches.

TIER 2 -- LoRA architecture (orthogonal to data choice, combine with Tier 1 data)
  target_full_attn    : LORA_TARGET_MODULES = ["q_proj","k_proj","v_proj","o_proj"]

  sysprompt_strict    : SYSTEM_PROMPT = "Follow ALL constraints exactly. Check word count, format, and forbidden words before responding."
  lr_1e4              : LEARNING_RATE = 1e-4

TIER 4 -- Advanced / experimental
  deltanet_layers     : WebSearch DeltaNet param names, then target them
  data_rejection_v2   : Re-run prepare_rejection.py --adapter after best adapter is found

  rejection_iterated  : Multiple rounds of rejection sampling with increasingly capable adapter

TIER 5 -- Last resort (confirmed to hurt)
  data_ultrafeedback  : Generic high-quality SFT -- less harmful than Alpaca but still weak signal
  data_alpaca         : CONFIRMED to reduce accuracy below baseline -- avoid
</exploration_tiers>

<decision_rules>

- If accuracy improves by >= 0.02: strong signal; explore nearby variations
- If Tier 1 data does not exist: run prepare_rejection.py and/or prepare_synthetic.py first
- If VRAM OOM: reduce BATCH_SIZE or MAX_SEQ_LEN; log in MEMORY_LOG.md
- Never change BUDGET_SECONDS
- Never edit evaluate.py, prepare*.py, db.py, log_result.py, or context_gen.py
- Generic SFT (Alpaca) is confirmed to hurt -- never use it unless explicitly testing
</decision_rules>
